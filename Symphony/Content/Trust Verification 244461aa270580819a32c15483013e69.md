# Trust Verification

## Overview

VSCode Marketplace uses trust indicators to help users identify reliable and high-quality extensions. These verification badges appear on extension pages and influence search rankings, user trust, and installation decisions.

## Types of Trust Verification

### üî∑ Microsoft Verified

**What it means:** Extension is published by Microsoft or a Microsoft-owned organization

**Who can get it:**

- Microsoft employees publishing official extensions
- Microsoft subsidiary companies (GitHub, LinkedIn, etc.)
- Official Microsoft product teams

**How to qualify:**

- Must be employed by Microsoft with valid corporate email
- Publisher account must be linked to Microsoft Azure AD tenant
- Extension must be published under official Microsoft publisher namespace
- Undergo internal Microsoft code review and security validation

**Application process:**

1. Contact VSCode Marketplace team through internal Microsoft channels
2. Provide proof of employment and team authorization
3. Submit extension for internal security and quality review
4. Complete Microsoft publisher onboarding process
5. Automatic verification upon approval

**Note:** This verification is not available to external developers and is reserved exclusively for Microsoft-published extensions.

---

### üíö Sponsor Verified

**What it means:** Developer is actively sponsored through GitHub Sponsors program

**Who can get it:**

- Individual developers or organizations with active GitHub Sponsors
- Publishers who demonstrate sustained community support through sponsorship

**Qualification criteria:**

- Active GitHub Sponsors profile with minimum $100/month in sponsorship
- Maintained sponsorship for at least 6 consecutive months
- GitHub account linked to VSCode publisher profile
- Sponsor tier must be "Active Contributor" level or higher

**How to achieve:**

1. **Set up GitHub Sponsors:**
    - Create GitHub Sponsors profile on your account/organization
    - Set up sponsorship tiers and goals
    - Add compelling description of your open source work
2. **Build sponsorship base:**
    - Promote your sponsors profile through documentation, README files
    - Engage with community through quality extensions and support
    - Share your work on social media and developer communities
    - Consider offering sponsor perks (early access, direct support, etc.)
3. **Link accounts:**
    - Connect your GitHub account to VSCode publisher profile
    - Ensure publisher email matches GitHub primary email
    - Enable GitHub integration in publisher settings
4. **Automatic verification:**
    - System checks GitHub Sponsors API weekly
    - Verification applies automatically once criteria are met
    - Badge appears within 7 days of qualifying

**Tips for success:**

- Focus on creating valuable, widely-used extensions
- Provide excellent documentation and user support
- Engage actively with your user community
- Share your development journey and challenges
- Collaborate with other developers and cross-promote

### ‚≠ê Community Verified

**What it means:** Extension has achieved high community trust through usage and ratings

**Who can get it:**

- Any publisher whose extensions demonstrate strong community adoption
- Extensions that maintain high quality and user satisfaction over time

**Qualification criteria:**

- **Install base:** Minimum 100,000 total installs
- **Rating quality:** Average rating of 4.0+ stars out of 5
- **Rating quantity:** At least 100 user ratings
- **User engagement:** 80%+ positive ratings (4-5 stars)
- **Unique reviewers:** Minimum 50 different users providing reviews
- **Maturity:** Extension published for at least 6 months
- **Maintenance:** Regular updates within past 3 months

**How to achieve:**

1. **Focus on extension quality:**
    - Solve real problems that developers face daily
    - Ensure robust functionality with minimal bugs
    - Provide comprehensive documentation and examples
    - Follow VSCode extension development best practices
2. **Build user base organically:**
    - Share your extension in relevant developer communities
    - Write blog posts or tutorials demonstrating use cases
    - Engage on social media (Twitter, LinkedIn, Reddit)
    - Present at meetups or conferences
    - Collaborate with other extension developers
3. **Maintain high user satisfaction:**
    - Respond promptly to user issues and questions
    - Implement frequently requested features
    - Provide clear installation and usage instructions
    - Keep extension compatible with latest VSCode versions
    - Regular bug fixes and performance improvements
4. **Encourage positive reviews:**
    - Implement unobtrusive prompts for satisfied users to rate
    - Provide excellent user experience that naturally leads to positive feedback
    - Address negative reviews constructively and quickly
    - Show appreciation for user feedback and contributions
5. **Sustain long-term growth:**
    - Monitor extension analytics and user feedback patterns
    - Iterate based on user needs and marketplace trends
    - Maintain consistent update schedule
    - Build extension ecosystem (related tools, companion extensions)

**Verification timeline:**

- Evaluated automatically every week
- Badge granted immediately upon meeting all criteria
- Re-evaluated monthly to ensure continued qualification

## Trust Signals That Influence Verification

### Repository Health

- **Active development:** Regular commits and updates
- **Documentation quality:** Comprehensive README, contribution guidelines
- **Community engagement:** GitHub stars, forks, contributor activity
- **Security practices:** No known vulnerabilities, security policy
- **License clarity:** Proper open source licensing

### Marketplace Performance

- **Install growth rate:** Steady, organic growth patterns
- **User retention:** Users who install continue using over time
- **Support responsiveness:** Quick response to reviews and issues
- **Update frequency:** Regular maintenance releases and feature updates

### Community Presence

- **Developer reputation:** Established presence in developer community
- **Social validation:** Mentions on social media, blogs, Stack Overflow
- **Professional background:** LinkedIn profile, portfolio, contribution history
- **Multi-platform presence:** Activity across GitHub, npm, other platforms

## Best Practices for All Developers

### Development Standards

- Follow VSCode Extension Guidelines and API best practices
- Implement comprehensive error handling and logging
- Ensure cross-platform compatibility (Windows, macOS, Linux)
- Optimize performance and minimize resource usage
- Provide localization support for international users

### Security Excellence

- Undergo regular security scans and address findings promptly
- Follow secure coding practices and input validation
- Implement proper permission scopes and user consent
- Maintain transparency about data collection and usage
- Respond quickly to security reports from community

### Community Engagement

- Maintain active presence in VSCode extension development community
- Participate in discussions, forums, and GitHub issues
- Provide helpful support to other developers
- Share knowledge through documentation, tutorials, or open source contributions
- Be responsive and professional in all community interactions

### Long-term Success

- Plan for extension lifecycle and maintenance commitments
- Build sustainable development practices
- Consider succession planning for popular extensions
- Maintain backwards compatibility when possible
- Communicate changes and deprecations clearly to users

## Common Misconceptions

### "Verification guarantees perfect security"

**Reality:** Verification indicates higher trust levels but doesn't eliminate all risks. Users should still exercise judgment and follow security best practices.

### "Only large companies can get verified"

**Reality:** Community verification is specifically designed for individual developers and small teams. Quality and community impact matter more than company size.

### "Verification is permanent once achieved"

**Reality:** All verification types are subject to ongoing evaluation. Poor maintenance, security issues, or community feedback can result in verification removal.

### "Non-verified extensions are unsafe"

**Reality:** Many excellent extensions exist without verification badges. Verification is one factor among many when evaluating extension quality.
